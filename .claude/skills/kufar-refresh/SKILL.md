# /kufar-refresh — проверка актуальности выгрузки

Перепроверяет сохранённые в прошлом отчёте объявления на kufar.by: цена, статус, наличие. Не запускает полный парс категорий — только точечно дёргает карточки из `reports/<тема>-подбор.json` и `.cache/seen_ids.json`.

## Когда применять

- Пользователь: «проверь актуальность», «обнови цены», «что-то ещё в продаже?», «сними с продажи то, чего нет».
- Не применять, если последний полный парс был > 24 часов назад — в этом случае предложить сначала `/kufar-upgrade` для свежего сбора.

## Что делает

1. Найти входной отчёт:
   - если в аргументе передано имя файла — взять его;
   - иначе — самый свежий `reports/*-подбор.md` (или `-pick.md`).
2. Прочитать пары `mb_id` / `cpu_id` из соответствующего `reports/<тема>-подбор.json`.
3. Для каждого id:
   - `WebFetch` или HTTP-GET `https://www.kufar.by/item/<id>`;
   - вытащить текущую `price` и наличие (статус «активно» / «снято»);
   - сравнить с прошлой ценой → дельта в %;
   - если объявления нет (404 / статус «удалено») — пометить `removed=true`, `removed_at=<now>`.
4. Обновить `.cache/seen_ids.json`: для активных — `last_seen=now`, для удалённых — `removed=true`.
5. **Не удалять** удалённые объявления из истории (правило CLAUDE.md).
6. Перегенерировать `reports/<тема>-подбор.md` и `.json` с пометками:
   - `removed=true` — отдельной секцией «Снято с продажи»;
   - `price_changed` — в топе указать новую цену и дельту;
   - `defect` — если в новом body появились дефект-маркеры — пометить и убрать из топа.
7. Если все топ-3 объявления ушли из выдачи — предложить запустить `/kufar-upgrade` для свежего сбора.

## Входные данные (аргумент)

- 0 аргументов: взять самый свежий `reports/*-подбор.md`.
- 1 аргумент: имя файла отчёта без расширения (например `2026-06-10 — intel+amd ddr4 upgrade — подбор`).

## Файлы

- Читает: `reports/<тема>-подбор.{md,json}`, `.cache/seen_ids.json`.
- Пишет: обновляет `reports/<тема>-подбор.{md,json}`, `.cache/seen_ids.json` (только `removed`/`last_seen`).
- **Не удаляет** записи из `seen_ids.json` (правило CLAUDE.md).
- **Не переименовывает** файлы (правило CLAUDE.md).

## Границы

- Не выходить за пределы рабочей папки.
- Не отправлять ничего на kufar.by.
- Не дёргать карточки объявлений, которых нет в текущем отчёте — `/kufar-refresh` точечный, не массовый.

## Что уточнять у пользователя перед прогоном

1. Какой отчёт проверять (если не передан аргумент и не очевидно из контекста).
2. Если прошло > 24 часов с последнего полного парса: «хотите сначала полный /kufar-upgrade, или точечно проверить только текущий топ?».
3. Что делать с объявлениями, ушедшими из топа (но не удалёнными с kufar): оставить в выгрузке как «запасные» или выкинуть. По умолчанию — оставить.

## Сценарий по умолчанию (план)

1. Найти входной отчёт.
2. Собрать список `mb_id` + `cpu_id`.
3. Для каждого: `WebFetch` `https://www.kufar.by/item/<id>`, вытащить цену/статус.
4. Сравнить с прошлыми значениями, посчитать дельту.
5. Обновить `seen_ids.json` (только `removed`/`last_seen`).
6. Перегенерировать отчёт с пометками.
7. Сводка из 3 пунктов: «что снято / что подорожало / что упало в цене».
