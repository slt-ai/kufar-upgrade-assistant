При детальном обходе страниц kufar.by задержка 0.5 с между запросами вызвала HTTP 429 на ~91 страницах. После увеличения задержки до 2.5 с и добавления до 3 retries с экспоненциальным backoff все 194 детальные страницы были собраны успешно.

**Why:** kufar.by замедляет/блокирует частые запросы; без достаточных пауз детальный сбор ломается.

**How to apply:** Для future crawls kufar-listings-fetcher использует KUFAR_ITEM_DELAY=2.5с (или выше) и retries при сборе detail-страниц. При первых признаках 429/пустых ответов — увеличивать задержку, а не пытаться пробить лимит.
