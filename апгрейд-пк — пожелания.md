# Пожелания по апгрейду ПК — единый источник параметров

**Этот файл + `Report.txt` — единственный источник параметров апгрейда.**
Главный агент, `kit-analyzer` и `kufar-listings-fetcher` читают их при каждом запуске
и **не хранят** эти значения в своих определениях. Правка файла → система ищет по новым
параметрам без правки агентов.

Структура:
- секции **2–3, 5–6, 7–8** — ваши пожелания (крутите под новый сценарий: регион, категории,
  DDR4/DDR5, бюджеты, тиры, приоритеты, ОЗУ);
- секция **4** — справочник совместимости (факты железа, агенты читают, вы редко правите);
- секция **1** — анкор из `Report.txt`.

Требования собраны из `CLAUDE.md`, `kit-analyzer.md`, `kufar-listings-fetcher.md` и ответов
пользователя (2026-07-06). Система задумана универсальной — заточенной под разные варианты
апгрейда (смена scope/региона/платформы = правка этого файла).

## 2. Жёсткие ограничения (не обсуждаются)

1. **Память: только DDR4.** DDR5 / AM5 / LGA1700-DDR5-ONLY — исключать.
2. **PCI-E:** цель **5.0 x16**, допустимый минимум **4.0 x16** на основном слоте под GPU.
   Если у платы несколько x16 — указывать, какой 4.0/5.0 (x4-слоты не считаются).
3. **Регион: Минск.** Лоты из других регионов — отбрасывать, не переранжировать.
4. **Whitelist брендов мат. плат:** ASUS, MSI, Gigabyte, ASRock, Biostar, ECS.
   Noname / unbranded — исключать.
5. **Defect-фильтр (negation-aware):** маркеры «не работает», «сломан», «дефект», «убит»,
   «не включается», «перестали работать», «вышел из строя», «разбит», «трещина», «глючит»
   (и близкие варианты) → `defect=true`, в подборку не пускать.
   При этом «без дефектов» / «исправен» / «нет дефектов» дефектом **не** считать.
6. **Мат. плата и CPU — из разных объявлений**, комбинируются в пары. Один лот может
   участвовать в нескольких парах, если совместим.
7. **«Договорная» / без цены** — пропускать на этапе фильтра: сумму и ранг посчитать нельзя.
8. **Свежесть:** окно 7 дней от `list_time`. Price-delta > **10%** или смена статуса →
   `price_changed=true` со старой/новой ценой.
9. **Removed:** объявление, отсутствующее в свежей выдаче, **не** помечать `removed=true`
   автоматически. Для ценных/подходящих лотов — проверка прямой ссылки
   `https://www.kufar.by/item/<id>`. Только 404/gone → `removed=true`. Без проверки →
   `stale=true, last_check=<now>`. Из истории не удалять.
10. **Читать `body`, а не только `title`/`params`** — модель CPU, дефекты, нюансы совместимости
    часто только в описании.

> `kufar-listings-fetcher` хранит backstop-копию пунктов 5 и 7 (defect-маркеры + no-price) —
> на случай отсутствия/порчи этого файла. Авторитетный список — здесь; backstop держать в синка.

## 4. Справочник совместимости (hardware facts — редко правится)

Факты железа: какой сокет какие поколения CPU и чипсеты принимает, какой тип памяти и PCIe.
`kit-analyzer` сверяется с этой таблицей при проверке пары плата+CPU. Правьте только если
добавляете новую платформу в scope.

| Сокет | Поколения CPU | Чипсеты | Память | PCIe | Примечания |
|-------|---------------|---------|--------|------|------------|
| LGA1700 | Intel 12/13/14 gen | Z690/B660/H610/Z790/B760 | DDR4 **или** DDR5 (вариант платы) | 4.0/5.0 | DDR5-only варианты платы — исключать по правилу 2.1 |
| LGA1200 | Intel 10/11 gen | Z490/B460/H410/Z590/B560 | DDR4 | 3.0/4.0 | 12th-gen CPU не становятся — reject |
| LGA1151 v2 | Intel 8/9 gen | Z390/B365/H310 | DDR4 | 3.0 | только 8/9 gen |
| AM4 | Ryzen 2000/3000/5000 | B450/X470/B550/X570 (A320/B450M и т.п.) | DDR4 | 3.0/4.0 | Ryzen 5000 на B450/X470 — **только** при упоминании обновлённого BIOS |
| AM5 | Ryzen 7000/9000 | B650/X670/B850 | **DDR5 only** | 4.0/5.0 | исключать по правилу 2.1 (DDR4-only) |

**Правила проверки (механика в `kit-analyzer.md`):**
- Socket match: LGA==LGA (номер), AM4==AM4, AM5==AM5. Mismatch → reject.
- CPU должен быть в списке поддерживаемых платой (чипсет/поколение из таблицы + note из `body`).
- Ambiguous `body`/`title` → `compatibility=uncertain`, демотировать ниже verified пар.
- Форм-фактор/БП — игнорировать, кроме явного VRM/TDP-мисматча (125W CPU на 4-фазной бюджетной
  плате) → флаг-нота, не exclude.

## 5. Тиры и бюджет

Три отдельных рекомендации **на каждую платформу** (Intel и AMD), не просто ценовые корзины.
Baseline = текущий CPU из `Report.txt` (i7-9750H = 100 pts; Passmark/Cinebench R23 relative).

| Тир | Смысл | CPU perf (pts) | Бюджет комплекта (BYN) |
|-----|-------|----------------|------------------------|
| `entry_upgrade` | Самый дёшевый апгрейд, **не хуже** текущего | ≥ 98 (≥ baseline) | без жёсткого верха, ранжировать по цене ↑ |
| `balanced` | **Максимум за минимальные деньги** — лучший perf/price | 115–300 | > 1.15× cheapest entry **и** ≤ max(1100, 2.2× entry) |
| `future_ready` | Мощно, но не безумно, **с заделом под апгрейд** | ≥ 180 | ≤ max(1500, 3× cheapest entry) |

## 6. Параметры скоринга (механика в `kit-analyzer.md`)

- **Baseline:** текущий CPU из `Report.txt` = 100 pts.
- **Пороги perf:** entry ≥ 98; balanced 115–300; future_ready ≥ 180.
- **Бюджетные cap:** balanced ≤ max(1100 BYN, 2.2× entry); future_ready ≤ max(1500 BYN, 3× entry);
  balanced строго > 1.15× cheapest entry (чтобы тиры не сливались).
- **VRM-мисматч (125W CPU на H610/B660):** в `balanced` умножать `perf/price` на **0.75**;
  в `future_ready` — сильный штраф (флагманский i9 на H610 не должен выиграть тир),
  но не exclude при дефиците premium-плат.
- **Веса `future_ready`:** perf **45%**, качество/будущестойкость платы **35%**
  (Z/X570, PCIe 5.0, 4 DIMM), цена **20%**.
- **Дедуп между тирами:** попавший в `entry_upgrade` combo в `balanced`/`future_ready` не повторяется.
- **Сохранять старый breakdown** (`price`, `compatibility`, `freshness`, `condition`, `seller`)
  рядом с меткой тира и `cpu_perf`.

## 7. Приоритеты под локальные LLM-модели

Контекст пользователя: машина — для запуска локальных LLM и нейросетей.

- **Генерация — на GPU** (RTX 5060 Ti сейчас; на CPU медленно). CPU не тянет инференс, поэтому
  CPU-перф — **вторичный** фактор; главный — дать видеокарте полноценную шину и задел под замену.
- **Задел под более мощную видеокарту в будущем.** Плата должна принимать GPU с запасом по полосе:
  **PCIe 5.0 x16 предпочтительнее 4.0**, даже если текущей карте хватает 4.0. Это явный бонус в
  `future_ready` и тай-брейк в `balanced`.
- **«Вытянуть максимум за минимальную сумму».** При прочих равных — дешевле.
  `balanced` — приоритетный тир под этот запрос.
- CPU-фичи (AVX-512/AMX, много ядер, ECC) — **не первичны** для генерации (она на GPU).
  Учитывать только как тай-брейк/бонус, не как фильтр. Не отбраковывать платформу только за
  отсутствие AVX-512.

## 8. ОЗУ — двухэтапный план

- **Этап 1:** оставить текущие **48 ГБ DDR4** (4 планки mix 3200/2400/2133). Плата должна иметь
  **4 слота DIMM** и терпеть mixed speeds.
- **Этап 2 (позже):** докупить до **64+ ГБ**. Поэтому 4-DIMM платы — в приоритете.
- **НО:** плату **с 2 слотами DIMM не отбрасывать**, если по остальным параметрам (чипсет,
  PCIe 5.0/4.0, VRM, бренд, состояние, цена) она — топ. В таком случае помечать
  `ram_slots=2, warns=["48GB в 4 планках не влезут — только 2×24 или пересборка"]` и не исключать.
