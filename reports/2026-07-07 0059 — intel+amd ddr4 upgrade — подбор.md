Intel больше perf за меньшие деньги; AMD больше потоков (32 vs 24) — лучше для LLM-инференса с большим батчем. **Под CPU-инференс LLM — AMD 5950X (32 потока), под универсальную нагрузку — Intel 13700F.**

Все рекомендуемые платы дают PCIe 4.0 x16 — допустимый минимум для RTX 5060 Ti. Если PCIe 5.0 критичен (§7 задел под будущую GPU) — нужен следующий прогон, расширение региона, либо допуск DDR5-платформы (тогда + замена RAM).

1. **2-DIMM в Intel entry** — у пользователя 4 планки DDR4. Intel #1 (H610M-HDV, 2 DIMM) помечен ⚠ → RAM даунгрейд до 32GB. Для сохранения 48GB брать Intel #2/3 (B760, 4 DIMM) или AMD (все 4 DIMM).
2. **Аудио на ASRock B550** (`1073248395`, AMD #1/#2) — аудиокодек отсутствует («отлетевшая звуковая карта»), плата рабочая. Если нужен встроенный звук → альтернатива Gigabyte B550 DS3H (`1075923437`, 250 BYN, 4 DIMM, рабочее аудио, прямая проверка — без дефектов).
3. **BIOS на ASRock B550 + Ryzen 5000** — для пары с Ryzen 5 5600 (alt к 3700X) уточнить версию BIOS у продавца (Ryzen 5000 out-of-box на актуальном BIOS). Для Ryzen 3000 (3700X, 4500) — без caveat.
4. **PCIe 5.0:** если критичен (§7 задел под будущую GPU) — ждать следующий прогон / расширить регион / допустить DDR5-платформу (тогда + замена RAM на DDR5). В текущем DDR4-пуле валидной whitelist-платы с 5.0 нет ни у Intel, ни у AMD.
5. Состав кита `1075770652` (MB+CPU+RAM @ 1600 BYN) — модели не раскрыты.

## Аномалии
- **Баг defect-флага (повтор):** `seen_ids.defect` не взводился при заполненном `defect_reason` / при частичном дефекте в body. Лот `1073399967` (3950X, «проблемы с 30,31 ядрами») ушёл бы в топ как валидный, если бы не ручная проверка body. Требует фикса в fetcher'е: defect = (defect_reason непустой) OR (body содержит дефект-маркеры, negation-aware). Связано с [[project-defect-bool-not-flipped]].
- **params.ram_type для Z790 надёжен:** гипотеза о ненадёжности не подтвердилась — params корректно показывал DDR5 для всех 4 проверенных Z790/B760-5.0 лотов. Гипотеза снята.
- **Biostar B550MH «3.0» = PCIe Gen3:** название модели содержит версию шины, не чипсет. B550 без Gen4 → исключать по pcie_too_old. Маркер для fetcher'а: «B550MH 3.0» / «3.0» в модели платы.
- **WebFetch из главного агента на kufar.by заблокирован** окружением («Unable to verify domain») — проверка карточек по правилу 9 делегирована fetcher'у (`scripts/_fetch_targeted_bodies.py`, urllib + 2.5c delay, 9/9 успешно 200 OK).
- **Цены** в `scripts/_out.json` в копейках (×100), в `scripts/_items_grouped.json` в BYN — нормализовано при суммировании.
- **Fetch-скрипт:** `scripts/kufar_parse.py` — `PAGE_DELAY` 0.5c → 2.5c (синк с rate-limit kufar.by: 429 при 0.5c, успех при 2.5c). Контракт не менялся.
