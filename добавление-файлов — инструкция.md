# Инструкция: как добавлять новый файл в проект

Правила добавления файлов в `D:\Work\AI\ClaudeCode\research-agent`. Соблюдать всегда — это проектные конвенции из `CLAUDE.md`.

## 1. Где размещать

| Тип файла | Папка | Примеры |
|-----------|-------|---------|
| Отчёты подбора, выгрузки, логи парсинга | `reports/` | `2026-07-04 — intel+amd ddr4 upgrade — подбор.md` |
| Скрипты, промежуточный вывод парсера, кеш | `scripts/`, `scripts/.cache/` | `kufar_parse.py`, `_out.json`, `seen_ids.json` |
| Входные данные / конфигурация ПК | корень проекта | `Report.txt` |
| Служебные документы (инструкции, шаблоны, чеклисты) | корень проекта | `добавление-файлов — инструкция.md` |

## 2. Имя файла

Формула: `[YYYY-MM-DD HHMM —] тема — тип.md`

- Разделитель — `—` (em dash **с пробелами** по бокам), не `-` и не `_`.
- Дата `YYYY-MM-DD` и время `HHMM` (24ч, без двоеточия) добавляются **вместе**, только когда файл привязан к конкретному прогону/дате (отчёты, выгрузки, логи). **Время `HHMM` обязательно при наличии даты** — в один день может быть несколько прогонов, без времени они перетрут друг друга.
- Служебные/шаблонные документы (инструкции, чеклисты) — **без даты и времени**.
- Тема — короткая, на латинице или кириллице, без слэшей и двоеточий.
- Тип — `подбор`, `выгрузка`, `лог`, `инструкция`, `чеклист` и т.п.

Примеры:
- `2026-07-04 2149 — intel+amd ddr4 upgrade — подбор.md` — отчёт подбора
- `2026-07-04 2149 — intel+amd ddr4 upgrade — подбор.json` — зеркальная выгрузка
- `client-onboarding-checklist.md` — чеклист (дата и время не нужны)

## 3. Зеркалирование форматов

Отчёт подбора всегда сохраняется в **двух-трёх** форматах с **одинаковым** базовым именем:

- `reports/<имя>.md` — человекочитаемый отчёт (обязателен)
- `reports/<имя>.json` — структурированная выгрузка топ-пар (обязателен)
- `reports/<имя>.csv` — таблица комплектов (опционально)

Базовое имя (`<дата — тема — тип>`) должно совпадать во всех трёх файлах. Меняется только расширение.

## 4. Safety-правила (что нельзя без явного одобрения)

- **Не удалять** существующие файлы.
- **Не переименовывать** существующие файлы.
- **Не выходить** за пределы рабочей папки `D:\Work\AI\ClaudeCode\research-agent`.
- **Не передавать** наружу личные данные пользователя.
- **Не отправлять** ничего на kufar.by (отклики, заявки продавцам).

Удалять/переименовывать можно только с явного одобрения пользователя.

## 5. Кодировка и содержимое

- Все текстовые файлы — UTF-8 без BOM.
- В отчётах подбора обязательно: краткая сводка (3 пункта), топ-3 Intel, топ-3 AMD, сравнительная таблица, секция «Изменения» относительно прошлого отчёта, список созданных/обновлённых файлов.
- В JSON-выгрузке — структурированные поля: `mb_id`, `cpu_id`, `price`, `seller`, `status`, `removed`, `defect`, `last_check` и т.п. (см. существующие `reports/*-подбор.json` как образец).

## 6. Проверка перед сохранением

1. Имя соответствует формуле `[YYYY-MM-DD HHMM —] тема — тип.md` (для date-bound файлов; время обязательно при наличии даты).
2. Файл лежит в правильной папке (`reports/` для отчётов, `scripts/` для промежуточных данных).
3. Если это отчёт подбора — есть зеркальный `.json` с тем же базовым именем.
4. Не затронуты/не удалены существующие файлы.
5. В конце ответа пользователю — список созданных/обновлённых файлов.
